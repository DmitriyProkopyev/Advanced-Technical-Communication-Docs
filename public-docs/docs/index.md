# Home

Welcome to the documentation guide written as lecture notes for **"Advanced Technical Documentation"** course.

> [!info] Scope
> Dynamicaly evolving software projects that require accurate documentation at all times, yet do not have a dedicated technical writer.

> [!people] Target Audience
> Developers who seek to quickly write a good enough documentation.

For a quick summary of the pages refer to the [guide](guide.md).

## Writing documentation from scratch

If you are writing from scratch, consider this order of operations:

1. Start by establishing the writing [processes](processes.md).
2. Decide and document the [voice and tone](voice_and_tone.md) of your documentation.
3. Design the [representation](representation.md) of the information you are communicating to the readers.
4. Revise the [formatting](formatting.md) of the text.

## Quick Fix

If you need immediate revision, run through the [formatting](formatting.md) checklist and optionally the [representation](representation.md) checklist.
